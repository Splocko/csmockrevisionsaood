1.1.1

Structure & function of the processor

The CPU is the brain of the computer: it fetches instructions from memory, decodes them and executes them, over and over, billions of times a second.

The components Y1

The processor is built from the ALU, the Control Unit, and a set of registers, connected to memory by three buses.

OCR Definition — ALU

Arithmetic and Logic Unit: the part of the processor that carries out all arithmetic calculations (add, subtract, multiply, shifts) and all logical operations (AND, OR, NOT, comparisons). It is also where the accumulator sits.

OCR Definition — Control Unit

Control Unit (CU): coordinates and controls the activities of the processor. It directs the flow of data around the CPU, decodes each instruction, and sends control signals to manage the fetch–decode–execute cycle.

The registers (learn these cold)

RegisterFull nameJob
PCProgram CounterHolds the address of the next instruction to be fetched. Increments each cycle (or is overwritten by a jump).
MARMemory Address RegisterHolds the address in memory that is about to be read from or written to.
MDRMemory Data RegisterHolds the data or instruction just fetched from memory, or about to be written to memory. (The "gateway" between CPU and memory.)
CIRCurrent Instruction RegisterHolds the current instruction being decoded and executed.
ACCAccumulatorStores the result of calculations performed by the ALU.

The three buses

A bus is a set of parallel wires that transfers data between components. The width of a bus = the number of bits it can carry at once.

  • Data bus — carries data and instructions between CPU and memory. Bidirectional.
  • Address bus — carries the address of the memory location to access. Unidirectional (CPU → memory). Its width sets the maximum addressable memory (n wires → 2n addresses).
  • Control bus — carries control signals/timing (e.g. read, write, interrupt). Bidirectional.

The Fetch–Decode–Execute cycle Y1

This is the continuous cycle the CPU repeats to run a program. Learn the register transfers — examiners love them.

// FETCH
[PC]      → MAR        address of next instruction copied to MAR
[MAR]     → address bus → memory
memory    → data bus → MDR   instruction fetched into MDR
[MDR]     → CIR        instruction copied to CIR
PC        → PC + 1     PC incremented, ready for next
// DECODE
Control Unit decodes the instruction held in the CIR
// EXECUTE
instruction carried out (e.g. ALU operates, result → ACC)

Factors affecting CPU performance Y1

  • Clock speed (Hz) — number of cycles per second. Higher clock = more instructions per second. A 3 GHz CPU has 3 billion cycles/sec.
  • Number of cores — each core can fetch–decode–execute independently, so more cores allow more instructions to be processed at the same time — but only if the software is written to use them (parallelisable).
  • Cache — small, very fast memory on/near the CPU holding frequently/recently used data and instructions, so the CPU avoids slow trips to RAM. More/larger cache (L1→L2→L3, L1 fastest & smallest) generally improves performance.
Exam trapDoubling the cores does not double speed: a single-threaded program can only run on one core, and there is overhead splitting/recombining tasks.

Pipelining Y2

OCR Definition

Pipelining: a technique where the processor begins fetching the next instruction while it is still decoding/executing previous ones — the stages of the FDE cycle overlap. This keeps all parts of the CPU busy and improves throughput.

Processor architectures Y1

OCR Definition — Von Neumann

Von Neumann architecture: uses a single memory and a shared bus for both instructions and data (the stored program concept). Simpler and cheaper, but the shared bus creates a bottleneck (the "von Neumann bottleneck").

OCR Definition — Harvard

Harvard architecture: uses separate memories and separate buses for instructions and data. Instructions and data can be fetched simultaneously — faster, used in embedded systems and DSPs, but more complex/expensive.

Contemporary processors are largely von Neumann but borrow Harvard ideas (e.g. separate L1 instruction and data caches) plus pipelining and multiple cores.

GPUs Y2

OCR Definition

GPU (Graphics Processing Unit): a processor with many cores specialised for performing the same operation on large amounts of data simultaneously (highly parallel arithmetic).

Originally for rendering graphics, GPUs are now used for any task with lots of parallel, repetitive calculations: machine learning / AI training, scientific simulations, image & video processing, and cryptocurrency mining.

Multicore & parallel systems Y1

  • Multicore — multiple complete processing cores on one chip; each can run a separate thread/process, enabling true parallel execution.
  • Parallel processing — multiple instructions carried out simultaneously, across multiple cores or multiple processors.
  • Limitation: the problem must be divisible into independent parts; not all tasks parallelise, and there is overhead managing the split.

1.2.1

Systems software Y1

Purpose of an operating system

OCR Definition

Operating System (OS): software that manages the hardware and resources of a computer and provides a platform on which application software can run.

Key OS jobs: memory management, processor/process scheduling, interrupt handling, file management, security/user accounts, providing a user interface, managing input/output and peripherals (via drivers), and providing utilities.

Memory management

Paging

Memory is divided into equal, fixed-size blocks called pages; physical memory is divided into frames of the same size. Pages can be swapped between RAM and disk. Fixed size makes management simple but can waste space (internal fragmentation).

Segmentation

Memory is divided into variable-size, logical blocks called segments that reflect the structure of the program (e.g. a whole function/module). Sizes vary.

Virtual memory

OCR Definition

Virtual memory: a section of secondary storage (e.g. the hard disk) used as if it were RAM. Pages not currently needed are moved out to disk, allowing programs larger than physical RAM to run.

Exam trapExcessive swapping of pages between RAM and disk is called disk thrashing — it slows the system because disk access is far slower than RAM.

Interrupts & the ISR

OCR Definition

Interrupt: a signal sent to the processor requesting its attention, generated by hardware or software, that may cause the CPU to temporarily stop what it is doing to deal with the event.

Interrupts have priorities. At the end of each FDE cycle the CPU checks the interrupt register. If a higher-priority interrupt is present:

  1. The contents of registers (the current state) are saved onto a stack.
  2. The relevant Interrupt Service Routine (ISR) is loaded and run.
  3. When finished, the saved state is restored (popped off the stack) and the interrupted program continues.

Scheduling

The OS decides which process gets CPU time and for how long, to maximise use and keep the system responsive. Know these five algorithms:

AlgorithmHow it works
Round RobinEach process gets a fixed time slice in turn; if not finished it goes to the back of the queue. Fair, but ignores priority/length.
First Come First Served (FCFS)Processes run in arrival order until complete. Simple, but a long job blocks everything behind it.
Shortest Job First (SJF)Runs the job with the shortest total time next. Needs to know job lengths in advance; long jobs may starve.
Shortest Remaining Time (SRT)Runs the job with the shortest remaining time; pre-emptive — a newly arrived short job can interrupt.
Multi-level Feedback QueuesSeveral queues with different priorities; processes move between queues based on behaviour. Flexible but complex.

Types of operating system

TypeKey feature
DistributedRuns across multiple networked computers, sharing the workload as one system.
EmbeddedBuilt for a specific device with limited resources (e.g. washing machine); efficient, limited functions.
Multi-taskingRuns several programs apparently at once using time-slicing.
Multi-userAllows multiple users to access one system, sharing resources (uses scheduling to allocate fairly).
Real TimeGuarantees a response within a strict time limit (e.g. aircraft control, pacemaker).

BIOS, device drivers & virtual machines

  • BIOS (Basic Input/Output System): the first program run when a computer is switched on. It runs the POST (power-on self-test) to check hardware, then loads the operating system. Stored in ROM/flash.
  • Device driver: software that allows the OS to communicate with and control a particular piece of hardware. OS-specific.
  • Virtual machine: software that takes on the function of a machine — any instance where software emulates hardware. Includes running an OS within another OS, or executing intermediate code (e.g. Java bytecode on the JVM). Benefits: portability, isolation/sandboxing. Drawback: slower than running natively.

  • Application software — programs that let a user do a specific task (word processor, browser, game).
  • Utility software — programs that help maintain/optimise the computer system: disk defragmenter, file management, backup, compression, antivirus, system clean-up.

Translators Y1

OCR Definition

Translator: a program that converts source code written by a programmer into machine code the processor can execute.

TranslatorWhat it does
CompilerTranslates the whole program into machine code in one go, producing an executable. Slow to compile, fast to run, errors reported all at end; runs without the source.
InterpreterTranslates and executes the program line by line. Easy to debug (stops at first error), platform-independent, but slower at run time and needs the interpreter present.
AssemblerTranslates assembly language into machine code (roughly 1 instruction → 1 machine instruction).

Stages of compilation Y2

  1. Lexical analysis — removes whitespace/comments and converts the source into a series of tokens; builds a symbol table of identifiers.
  2. Syntax analysis — checks tokens against the rules (grammar) of the language; builds a parse/abstract syntax tree; reports syntax errors.
  3. Code generation — produces the object (machine) code equivalent of the program.
  4. Optimisation — makes the code run faster and/or use fewer resources (e.g. removing redundant instructions).

Linkers, loaders & libraries Y2

  • Library — pre-written, pre-compiled, tested code (modules) that can be reused in programs.
  • Linker — combines the compiled program with any library modules it uses into one executable. Static linking copies code in; dynamic linking connects at run time (DLLs).
  • Loader — part of the OS that copies the executable from disk into memory and prepares it to run.

Types of programming language

Programming paradigms Y2

OCR Definition

Programming paradigm: a style or approach to programming. Different problems suit different paradigms.

  • Procedural — instructions executed in sequence, grouped into procedures/functions (Python, C, VB).
  • Object-oriented — program modelled as interacting objects (see OOP section).
  • Low-level / assembly — closely matches the processor's machine instructions.

Procedural languages Y1

Programs are sequences of instructions, using the constructs sequence, selection and iteration, organised into reusable procedures and functions. Imperative: you tell the computer how to do something step by step.

Assembly language & the Little Man Computer Y1

Assembly is a low-level language using mnemonics (e.g. ADD, STA) with (typically) one assembly instruction per machine instruction. The Little Man Computer (LMC) is OCR's teaching instruction set.

Programming techniques

The three constructs Y1

  • Sequence — instructions executed one after another, in order.
  • Selection (branching) — a choice between paths using if/elseif/else or switch/case.
  • Iteration — repetition: count-controlled (for) or condition-controlled (while, do…until).

Recursion: a subroutine that calls itself. It must have a base case (a stopping condition) and must move towards it, or it will never terminate.

Object-Oriented Programming Y2

OOP models a problem as a set of interacting objects, each bundling its own data and the operations on that data.

The core vocabulary

OCR Definition — Class

Class: a template/blueprint for creating objects. It defines the attributes and methods that its objects will have.

OCR Definition — Object

Object: an instance of a class, with its own state (attribute values) and behaviours.

OCR Definition — Attribute

Attribute: a variable belonging to a class/object that stores its data/state.

OCR Definition — Method

Method: a subroutine (function/procedure) belonging to a class that defines its behaviour.

Encapsulation

OCR Definition

Encapsulation: keeping an object's attributes private and only allowing access through public methods (getters/setters). This protects data from invalid changes and hides internal detail.

Inheritance

OCR Definition

Inheritance: a (sub/child) class inherits the attributes and methods of a (super/parent) class, then can add to or override them. Promotes code reuse.

Polymorphism

OCR Definition

Polymorphism: objects of different (sub)classes can be treated as objects of a common parent class, with each responding to the same method call in its own way (e.g. each subclass overrides speak()).

OCR pseudocode you must recognise
class Pet
  private name
  public procedure new(givenName)
    name = givenName
  endprocedure
endclass

class Dog inherits Pet
  private breed
  public procedure new(givenName, givenBreed)
    super.new(givenName)   // call parent constructor
    breed = givenBreed
  endprocedure
endclass

myDog = new Dog("Fido", "Terrier")   // create an object
Remember the keywordsConstructor = procedure named new · inheritance = inherits · call parent = super. · access = public/private · create object = new ClassName(...).

Why use OOP? (benefits)

  • Reusability — classes & inheritance reuse code.
  • Maintainability — encapsulation localises changes.
  • Modularity — objects are self-contained, easier to test & develop in teams.

Compression, encryption & hashing

Compression Y1

LossyLossless
Permanently removes some data; original cannot be fully recovered.No data lost; original is perfectly reconstructed.
Much smaller files. Used for images/audio/video (JPEG, MP3) where small quality loss is acceptable.Less reduction. Used where every bit matters (text, ZIP, PNG, software).

Lossless methods Y2

  • Run Length Encoding (RLE) — replaces runs of repeated values with one value + a count. Great for data with long repeats (e.g. simple images); useless if little repetition. AAAAABBB → 5A3B
  • Dictionary coding — frequently-occurring patterns are stored once in a dictionary; the data is replaced by shorter references (indexes) to the dictionary, which is sent with the file.

Encryption Y2

SymmetricAsymmetric
The same key encrypts and decrypts.A key pair: a public key encrypts, the matching private key decrypts.
Fast, but the key must be shared securely (the key-distribution problem).No need to share a secret key → solves key distribution; slower. Underpins HTTPS & digital signatures.

Hashing Y2

OCR Definition

Hashing: applying an algorithm to an input to produce a fixed-size output (the hash). It is one-way — you cannot reverse the hash to get the original input.

Uses: storing passwords securely (store the hash, not the password); fast lookup in a hash table (the hash gives the storage index); checking data integrity. A good hash function minimises collisions (two inputs giving the same hash).

Databases

Foundations Y1

  • Flat file — a single table/file; simple but causes data redundancy & inconsistency.
  • Relational database — data split across linked tables (entities); reduces redundancy.
  • Primary key — uniquely identifies each record in a table.
  • Foreign key — a primary key from one table used in another to link the tables.
  • Secondary key — an alternative field used for fast searching/sorting (an index on a non-key field).

Entity relationship (ER) modelling

Capturing, selecting, managing & exchanging data Y1

Capture: forms, OCR, OMR, sensors. Select: Query By Example, SQL. Manage: add/edit/delete/process. Exchange: common formats like CSV, JSON, XML.

Normalisation to 3NF Y2

OCR Definition

Normalisation: organising data in a database to reduce redundancy and improve integrity.

FormRule
1NFNo repeating groups; each cell holds a single (atomic) value; each record unique.
2NFIn 1NF and no partial dependencies — every non-key attribute depends on the whole primary key.
3NFIn 2NF and no non-key attribute depends on another non-key attribute (no transitive dependencies).
One-liner for the exam3NF = "every non-key attribute depends on the key, the whole key, and nothing but the key."

SQL — interpret & modify Y2

-- select
SELECT Name, Grade FROM Student
WHERE Grade > 60
ORDER BY Name ASC;

-- join two tables
SELECT Customer.Name, Order.Total
FROM Customer
INNER JOIN Order ON Customer.ID = Order.CustomerID;

INSERT INTO Student (Name, Grade) VALUES ("Aisha", 72);
UPDATE Student SET Grade = 80 WHERE Name = "Aisha";
DELETE FROM Student WHERE Grade < 40;

Referential integrity Y2

OCR Definition

Referential integrity: ensuring that a foreign key value always refers to an existing, valid record in the related table — so you can't reference data that doesn't exist.

Transactions & ACID Y2

OCR Definition

Transaction processing must satisfy ACID:

A — AtomicityA transaction is all-or-nothing; if any part fails, the whole thing is rolled back.
C — ConsistencyA transaction leaves the database in a valid state, obeying all rules.
I — IsolationConcurrent transactions don't interfere; results are as if run one at a time.
D — DurabilityOnce committed, changes are permanent, even after a crash/power loss.

Record locking stops two users editing the same record at once (preventing lost updates); redundancy (backups/replication) protects against data loss.

Web technologies

HTML, CSS & JavaScript Y1

  • HTML — defines the structure/content of a web page (headings, paragraphs, links, images).
  • CSS — defines the style/presentation (colours, fonts, layout); keeps styling separate & reusable.
  • JavaScript — adds interactivity/behaviour, run client-side in the browser.

Search engine indexing Y2

OCR Definition

Search engine indexing: the process of crawling web pages (following links with "spiders"/bots) and storing key information (keywords, meta-data, links) in an index so results can be returned quickly.

PageRank algorithm Y2

OCR Definition

PageRank: an algorithm that ranks pages by importance, based on the number and quality of inbound links. A link from a high-ranking page counts for more than one from a low-ranking page.

Key idea: a page's rank is shared out among the pages it links to. More incoming links from important pages → higher PageRank → higher in results.

Server-side vs client-side processing Y2

Client-side (e.g. JavaScript)Server-side (e.g. PHP)
Runs in the user's browser; fast response, reduces server load; but code is visible & depends on the device.Runs on the web server; secure (code hidden), can access databases; but adds server load & needs a round-trip.

Binary, hex & conversion Y1

Place values (8-bit): 128 64 32 16 8 4 2 1. Hex groups binary into nibbles (4 bits).

Worked — denary → binary → hex

77 = 64 + 8 + 4 + 1 → 0100 1101
Group in 4s: 0100 = 4, 1101 = 13 = D → hex 4D

Negative numbers Y1

Sign & magnitude

Leftmost bit = sign (0 = +, 1 = −), remaining bits = magnitude. 1000 0101 = −5. Problem: two zeros (+0 and −0) and arithmetic is awkward.

Two's complement (preferred)

Worked — represent −5 in 8 bits

+5 = 0000 0101 → flip all bits → 1111 1010 → add 1 → 1111 1011

The leftmost place value is negative: −128 64 32 16 8 4 2 1. Check: −128+64+32+16+8+2+1 = −5 ✓

Binary addition & subtraction Y1

Worked — subtraction by adding the two's complement 
  9 - 5  =  9 + (-5)

   0000 1001   ( 9)
 + 1111 1011   (-5)
 -----------
 1 0000 0100   carry out of the 8th bit is DISCARDED
   ---------
   0000 0100   = 4  ✓
OverflowIf a carry changes the sign bit incorrectly (result won't fit in the bits available), that's overflow — the answer is wrong.

Floating point: representation & normalisation Y1

A real number is stored as a mantissa (the digits) and an exponent (where the binary point goes), both in two's complement in the OCR exam.

OCR Definition — Normalisation

Normalised floating point: the mantissa is adjusted so it starts with 0.1… for a positive number, or 1.0… for a negative number (the first two bits differ). This gives the maximum precision for the bits available.

Floating point arithmetic Y2

To add/subtract: align the exponents (shift the smaller number's mantissa so both exponents match), add/subtract the mantissas, then re-normalise the result.

Bitwise manipulation & masks Y2

  • Logical shift left/right — bits move; zeros fill in. Shift left by 1 ≈ ×2; shift right by 1 ≈ ÷2.
  • Mask — a binary pattern combined with data using AND/OR/XOR to read or change specific bits.
Worked — masking with AND/OR/XOR 
Data:   1011 0110
AND 0000 1111  → 0000 0110   isolate (keep) the low nibble
OR  0000 1111  → 1011 1111   set the low nibble to 1s
XOR 1111 1111  → 0100 1001   flip every bit

Karnaugh maps Y1

A K-map simplifies a Boolean expression visually. Plot 1s, then group adjacent 1s in blocks of powers of two (1,2,4,8…). Groups wrap around edges. Bigger groups = simpler terms. (Gray-code column/row order: 00, 01, 11, 10.)

Adders & the D-type flip-flop Y2

OCR Definition — Half adder

Adds two single bits, outputting a Sum (A ⊕ B) and a Carry (A · B). It cannot accept a carry in.

OCR Definition — Full adder

Adds two bits plus a carry-in, giving Sum and Carry-out. Full adders chain together to add multi-bit numbers.

OCR Definition — D-type flip-flop

A edge-triggered memory circuit that stores one bit. On the clock pulse (edge), output Q takes the value of input D and holds it until the next pulse. Flip-flops are the building blocks of registers and counters.
